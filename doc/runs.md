## Runs
The
[sjef projects](https://github.com/molpro/sjef/blob/master/README.md)
that iMolpro manages consist of an input, and perhaps other, files that specify the Molpro job to be run, together with any number of *Run* bundles, which are themselves structured as sjef projects.  A run contains all of the files (output and other) generated by a previous execution of Molpro; so in a working cycle where the job is executed, the input edited, and the job rerun, all of the history is retained.

iMolpro presents the current working input in its left hand pane, and on the right displays the results of a single Run, which by default is the most recent. The `Runs` menu can be used to access previous runs in a variety of ways:

- `Runs / Show Run...` supports selecting a different Run for display.
- `Runs / Open Run as Project...` enables the opening of a Run as a project in a separate window. A typical use would then be to `Projects / Move to...` the Run to a location in the file system that is outside of the current project.
- `Runs / Erase Run...` deletes a Run.
- `Runs / Adopt input from Run...` takes the input file from a previous Run as the current working input, overwriting the existing file.
